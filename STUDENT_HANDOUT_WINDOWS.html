Student Handout — Windows (Hands-on MLOps with Local CI/CD)

PowerShell for Python and Docker where possible; Git Bash for bundled .sh scripts. Same lab as the Linux/macOS handout, adapted for Windows paths and shells.

Goal

Build and deploy an ML model end-to-end: data generation → data security cleaning → train → real-time API → CI/CD → production deploy.

Windows prerequisites
  • Windows 10/11, 64-bit.
  • Python 3.10+ from python.org — enable Add python.exe to PATH (or use the py launcher).
  • Git for Windows (includes Git Bash) for running scripts/*.sh.
  • Docker Desktop with the WSL 2 or Hyper-V backend; start Docker before build/deploy steps.
If PowerShell blocks Activate.ps1, run once: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
0) Setup (host)

In PowerShell, go to your clone folder (name matches your clone, e.g. mlops-student-workshop):

PowerShell
cd "C:\path\to\mlops-student-workshop"
py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip
pip install -r requirements.txt

Git Bash alternative (same as Linux handout paths use /):

Git Bash
cd "/c/path/to/mlops-student-workshop"
python3 -m venv .venv
source .venv/Scripts/activate
pip install -U pip
pip install -r requirements.txt
1) Generate synthetic data

With the venv activated (PowerShell or Git Bash):

shell
python scripts/generate_synthetic_data.py --rows 5000 --seed 42 --out data/raw.csv

Data security note:

  • data/raw.csv contains PII columns: email, phone, customer_id.
2) Data cleaning + data security
shell
python src/data_prep.py --input data/raw.csv --output data/clean.csv

Expected outcome:

  • src/data_prep.py drops PII columns before training.
3) Train model + create artifacts

PowerShell — clear previous artifacts, then train:

PowerShell
if (Test-Path artifacts) { Remove-Item -Recurse -Force artifacts\* }
python src/train.py --train data/clean.csv --out artifacts --seed 42

Git Bash:

Git Bash
rm -rf artifacts/*
python src/train.py --train data/clean.csv --out artifacts --seed 42

You should see:

  • artifacts/model.joblib
  • artifacts/metrics.json
  • artifacts/metadata.json
4) Deploy to local “production” container

Recommended — Git Bash (runs the provided pipeline script):

Git Bash
./scripts/pipeline_local_deploy.sh

PowerShell — same end-to-end flow without Bash (Docker must be running):

PowerShell
python scripts/generate_synthetic_data.py --rows 5000 --seed 42 --out data/raw.csv
python src/data_prep.py --input data/raw.csv --output data/clean.csv
if (Test-Path artifacts) { Remove-Item -Recurse -Force artifacts\* }
python src/train.py --train data/clean.csv --out artifacts --seed 42
$tag = "mlops-workshop-student:local-$(Get-Date -Format 'yyyyMMdd-HHmmss')"
docker build -t $tag .
docker rm -f ml-prod 2>$null; docker run -d --name ml-prod -p 8000:8000 $tag

Verify (use curl.exe on Windows so you don’t hit PowerShell’s curl alias):

PowerShell / CMD
curl.exe -s http://localhost:8000/health
curl.exe -s http://localhost:8000/version
5) Prediction (real-time demo)
PowerShell / CMD
curl.exe -s -X POST http://localhost:8000/predict -H "Content-Type: application/json" -d "{\"age\":42,\"country\":\"US\",\"device_type\":\"mobile\",\"transaction_amount\":250.5,\"hour_of_day\":13,\"num_prev_txns\":3,\"is_international\":0}"

Git Bash — you can use the same JSON as the Linux handout with single quotes around the -d payload.

6) CI/CD practice (GitHub copy simulation)

Run the GitHub-style pipeline locally from Git Bash:

Git Bash
chmod +x scripts/github_pipeline_sim.sh
./scripts/github_pipeline_sim.sh

Then re-check:

PowerShell / CMD
curl.exe -s http://localhost:8000/version

The model_version should change after redeploy.

If pytest is not found in Git Bash, use python -m pytest -q after activating the venv.
7) Jenkins (if instructor starts local Jenkins)

Full setup (start Jenkins, job config, troubleshooting): follow jenkins/README.md in this repo.

Start Jenkins locally (PowerShell) from repo root:

PowerShell
cd jenkins
docker compose -f docker-compose.yml up --build -d

Create the Pipeline job: Pipeline script from SCM → Git → https://github.com/bridgeneuron-cloud/mlops-student-workshop.git → Branch */main → Script Path Jenkinsfile → Credentials (Username with password = GitHub username + PAT).

Expected stages (full pipeline): Setup Python + Dependencies → Generate synthetic data → Data prep → Train → Unit tests → Docker preflight → Build Docker image → Deploy to local production.

If Docker fails in Jenkins: set SKIP_DOCKER=true on the job to stop after tests. On your Windows host, run the deploy path from section 4 (Git Bash ./scripts/pipeline_local_deploy.sh or the PowerShell block).

After deploy: curl.exe -s http://localhost:8000/health and curl.exe -s http://localhost:8000/version.
